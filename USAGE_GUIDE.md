# 📘 二選一淘汰賽遊戲 - 使用指南

## 🎯 快速開始

### 第一步：準備圖片（可選）

#### 圖片建議

| 項目 | 建議值 | 說明 |
|------|--------|------|
| **尺寸** | 500x500 至 1000x1000 | 正方形最佳，系統會自動壓縮至 500x500 |
| **檔案大小** | < 1MB | 每張圖片建議小於 1MB |
| **格式** | JPG, PNG, WebP | 都支援，系統會統一轉為 JPG |
| **數量** | 1-5 張最佳 | 圖片越少，分享連結越短 |

#### 圖片來源建議

1. **免費圖庫**
   - [Unsplash](https://unsplash.com) - 高品質免費照片
   - [Pexels](https://pexels.com) - 免費圖片和影片
   - [Pixabay](https://pixabay.com) - 免費圖片庫

2. **自己的照片**
   - 手機拍攝（建議正方形裁切）
   - 電腦截圖
   - 設計軟件創作

3. **AI 生成圖片**
   - DALL-E
   - Midjourney
   - Stable Diffusion

### 第二步：編輯遊戲

#### 1. 上傳圖片（三種方式）

**方式 A：點擊上傳**
1. 點擊灰色預覽區域
2. 選擇圖片文件
3. 系統自動壓縮並顯示

**方式 B：拖放上傳** ⭐ 推薦
1. 從檔案管理器拖動圖片
2. 放到預覽區域上
3. 立即完成上傳

**方式 C：無圖片模式**
- 直接填寫文字即可
- 分享時選「僅文字」模式
- URL 最短，絕對不會出錯

#### 2. 編輯文字

- 在每個選項下方的文字框輸入
- 建議簡短有力（2-6 個字）
- 例如：溫婉、威嚴、可愛、強壯

#### 3. 管理選項

- **新增**：點擊「+ 新增選項」按鈕
- **刪除**：點擊選項右上角的「×」按鈕
- **最少**：至少保留 2 個選項才能玩遊戲

#### 4. 保存進度

**本地保存** 💾
```
點擊「保存草稿」→ 保存到瀏覽器本地
點擊「載入草稿」→ 載入之前保存的內容
```

**注意事項：**
- 本地保存使用 localStorage
- 清除瀏覽器數據會刪除草稿
- 不同瀏覽器的草稿不互通

### 第三步：分享遊戲

#### 分享選項對比

| 模式 | 包含內容 | URL 長度 | 適用場景 |
|------|----------|----------|----------|
| **包含圖片和文字** | 圖片 + 文字 | 2000-3000 字符 | 1-5 張圖片 |
| **僅文字** ⭐ | 僅文字 | 300-500 字符 | 6+ 張圖片或需要短連結 |

#### 分享步驟

1. **點擊「分享遊戲」按鈕**

2. **選擇分享模式**
   - 少量圖片：選「包含圖片和文字」
   - 大量圖片：選「僅文字」
   - 查看連結長度指示

3. **複製連結**
   - 點擊「複製」按鈕
   - 或手動選擇並複製

4. **生成 QR 碼**（可選）
   - 點擊「生成 QR 碼」
   - 手機掃描即可訪問
   - ⚠️ URL 超過 2000 字符建議先切換到「僅文字」

#### 分享途徑

- **社交媒體**：Facebook、Instagram、Twitter
- **即時通訊**：WhatsApp、Telegram、Line、微信
- **Email**：直接貼上連結
- **QR 碼**：截圖分享或打印

## 🎮 遊戲玩法

### 開始遊戲

1. 確保至少有 2 個選項
2. 點擊「開始遊戲」按鈕
3. 系統會隨機抽取兩個選項進行對比

### 遊戲規則

1. **二選一**
   - 每輪顯示兩個選項
   - 點擊你喜歡的那個
   - 未選中的會被淘汰

2. **持續淘汰**
   - 選中的選項保留
   - 新的選項補充進來
   - 繼續進行下一輪對比

3. **最終勝利**
   - 當只剩一個選項時遊戲結束
   - 顯示最終勝出的選項

### 遊戲操作

- **重新開始**：點擊「重新開始」重新玩一次
- **再玩一次**：結果頁面點擊「再玩一次」
- **返回編輯**：隨時可以返回編輯模式

## 🔧 故障排除

### 常見問題

#### Q: 分享連結打開後顯示「無法載入分享的遊戲數據」

**原因：**
- URL 太長被截斷
- 複製時沒有完整複製

**解決方案：**
1. 使用「僅文字」模式重新分享
2. 確保完整複製整個連結
3. 使用 QR 碼分享（避免截斷）

#### Q: 圖片上傳失敗

**可能原因：**
- 圖片檔案損壞
- 格式不支援
- 檔案過大

**解決方案：**
1. 嘗試其他圖片
2. 確保格式為 JPG/PNG/WebP
3. 壓縮圖片後再試

#### Q: 本地草稿無法載入

**可能原因：**
- 瀏覽器數據被清除
- 使用了不同的瀏覽器
- localStorage 不支援

**解決方案：**
1. 確保使用相同瀏覽器
2. 檢查瀏覽器設定是否允許 localStorage
3. 重新編輯並保存

#### Q: QR 碼生成失敗

**原因：**
- URL 太長（超過 2000 字符）
- QRCode 庫加載失敗

**解決方案：**
1. 切換到「僅文字」模式
2. 刷新頁面重試
3. 檢查網絡連接

## 📊 技術限制

### URL 長度限制

| 瀏覽器 | 最大 URL 長度 | 建議長度 |
|--------|--------------|----------|
| Chrome | ~2MB | < 2000 字符 |
| Firefox | ~65,000 字符 | < 2000 字符 |
| Safari | ~80,000 字符 | < 2000 字符 |
| Edge | ~2MB | < 2000 字符 |

**本應用建議：**
- 包含圖片：< 2000 字符
- 僅文字：通常 < 500 字符
- QR 碼：< 2000 字符

### 本地存儲限制

- **localStorage 限制**：每個網域 5-10MB
- **圖片數量**：建議 < 20 張
- **保存失敗**：減少圖片數量或移除圖片

## 🎨 自訂與擴展

### 修改預設選項

編輯 `app.js` 的 `DEFAULT_OPTIONS`：

```javascript
const DEFAULT_OPTIONS = [
    { text: '你的選項 1', image: null },
    { text: '你的選項 2', image: null },
    // ... 添加更多
];
```

### 調整圖片壓縮

修改 `compressImage` 函數參數：

```javascript
// 更高品質但更大的檔案
compressImage(file, maxWidth = 800, maxHeight = 800, quality = 0.8)

// 更激進的壓縮
compressImage(file, maxWidth = 300, maxHeight = 300, quality = 0.3)
```
