# 🎮 二選一淘汰賽遊戲

一個互動式的二選一淘汰賽遊戲，讓你可以自訂選項並進行有趣的比較遊戲。

## ✨ 功能特點

- **📝 自訂編輯器**：可以自由添加、編輯和刪除選項
- **🖼️ 圖片支援**：每個選項都可以上傳自訂圖片
- **🎯 淘汰賽機制**：通過二選一的方式逐步淘汰，直到選出最終勝者
- **🔗 分享功能**：通過 URL 分享你的遊戲設定（圖片和文字都會編碼在連結中）
- **📱 響應式設計**：支援桌面和移動設備

## 🎯 預設選項

## 🚀 使用方法

### 編輯模式

1. 點擊圖片區域上傳自訂圖片
2. 在文字框中輸入或修改選項文字
3. 使用「×」按鈕刪除不需要的選項
4. 點擊「+ 新增選項」添加更多選項

### 開始遊戲

1. 確保至少有 2 個選項
2. 點擊「開始遊戲」按鈕
3. 在每一輪中，點擊你喜歡的選項
4. 被選中的選項會保留，另一個會被淘汰
5. 繼續選擇，直到剩下最後一個選項

### 分享遊戲

1. 在編輯模式下，點擊「分享遊戲」按鈕
2. 系統會生成包含所有選項數據的連結
3. 複製連結分享給朋友
4. 朋友打開連結後，會自動載入你設定的所有選項和圖片

## 🛠️ 技術特點

- **純前端實現**：使用 HTML、CSS 和原生 JavaScript
- **智能壓縮**：使用 LZ-String 壓縮算法大幅減小分享連結長度
- **圖片優化**：自動壓縮上傳的圖片（最大 800x800，80% 品質）
- **URL Hash 存儲**：使用 URL hash (#) 而非 query parameter，支援更長的數據
- **無需後端**：所有數據都存儲在 URL 中
- **美觀動畫**：流暢的過渡和互動效果
- **錯誤處理**：完善的錯誤提示和異常處理

## 💡 技術說明

### URL 編碼機制（改進版）

遊戲使用高效的壓縮方式將數據編碼到 URL：

1. 將所有選項（包含文字和壓縮後的圖片）序列化為 JSON
2. 使用 **LZ-String** 壓縮算法進行壓縮（可減少 50-80% 的數據大小）
3. 將壓縮後的數據存儲在 URL hash (#) 中
4. 接收者訪問連結時，自動解壓縮並載入遊戲設定

**優勢：**
- URL hash 支援更長的數據（不受傳統 query parameter 限制）
- LZ-String 壓縮大幅減小連結長度
- 更好的瀏覽器兼容性

### 圖片處理與優化

- 使用 FileReader API 讀取上傳的圖片
- **自動壓縮**：調整圖片大小至最大 800x800 像素
- **品質優化**：JPEG 品質設為 80%，在視覺效果和檔案大小間取得平衡
- 轉換為 base64 格式存儲
- 分享時壓縮後的圖片會包含在 URL 中

**圖片優化效果：**
- 原始 2MB 圖片 → 壓縮後約 50-100KB
- 大幅減少分享連結長度
- 保持良好的視覺品質
